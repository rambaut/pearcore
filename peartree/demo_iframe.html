Ebola Virus Genomic Diversity

A phylogenetic analysis of 1610 Ebolavirus genomes — PearTree embedded viewer demo

Introduction

The 2013–2016 West African Ebola epidemic was the largest recorded outbreak of Ebolavirus disease, generating unprecedented amounts of genomic sequencing data. The phylogenetic tree below shows the evolutionary relationships among 1610 Ebola virus genomes sampled during the outbreak.

This page demonstrates embedding PearTree as a read-only tree viewer inside a report. The viewer runs entirely in the browser — no server required. You can pan and zoom, hover over tips, and drag your own tree file into the viewer.

Phylogenetic tree

Drag and drop a Newick, Nexus, or phyloXML tree file onto the viewer to load your own data.

How to embed PearTree

Drop peartree.html and its companion js/, css/ and vendor/ directories alongside your page, then add an <iframe>: 

<iframe
  src="peartree/peartree.html?palette=0&rtt=0&dt=0&import=0&export=0&nostore=1
       &treeUrl=https://example.com/my-tree.nwk"
  width="100%" height="600"
  style="border:none"
  title="Phylogenetic tree"
></iframe>

URL parameters

ParameterEffect
palette=0Hide the visual settings sidebar button
rtt=0Hide the root-to-tip divergence panel
dt=0Hide the data table panel
import=0Hide the Open Tree and Import Annotations buttons
export=0Hide the Export Tree and Export Graphic buttons
statusbar=0Hide the status bar
nostore=1Do not read or write settings to localStorage (avoids polluting the full PearTree app settings)
storageKey=my-keyUse a custom localStorage key (embed instances stay independent of each other)
treeUrl=https://…Fetch and load a tree from this URL on startup (http/https only)

Applying initial settings

Set window.peartreeConfig on the iframe's contentWindow before the module script runs, or use the pt:applyTheme postMessage after load: 

// Option A — configure before load (same-origin pages only)
const frame = document.getElementById('pt-frame');
frame.addEventListener('load', () => {
  // peartree fires 'peartree-ready' once fully initialised
  frame.contentWindow.addEventListener('peartree-ready', () => {
    frame.contentWindow.peartree.applyTheme('Dark');
  });
});

// Option B — postMessage (works cross-origin within same site)
frame.addEventListener('load', () => {
  frame.contentWindow.postMessage({ type: 'pt:applyTheme', name: 'Dark' }, '*');
});

// Option C — window.peartreeConfig set before the module runs
//            (works when the page and peartree.html are the same document)
window.peartreeConfig = {
  ui: { palette: false, rtt: false, dataTable: false, import: false, export: false },
  settings: { theme: 'Dark', branchColor: '#aaaaaa' },
  storageKey: null,   // no localStorage persistence
};

Programmatic tree loading via postMessage

// Fetch a tree and push it into the embedded viewer
const resp = await fetch('my-tree.nwk');
const text = await resp.text();
document.getElementById('pt-frame').contentWindow.postMessage(
  { type: 'pt:loadTree', text, filename: 'my-tree.nwk' },
  window.location.origin   // restrict to same origin
);

postMessage API

Message typePayloadEffect
pt:loadTree{ text, filename? }Parse and render text as a tree
pt:applyTheme{ name }Apply a named theme

Messages are only accepted from the same origin (or null for file:// contexts). Restrict the targetOrigin argument of postMessage appropriately.
